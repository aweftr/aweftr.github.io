交我算使用方法

很多同学反映交我算跑不起来代码，这里介绍跑代码的一些简单方式。

交大 HPC 目前我们经常使用的集群是：“思源一号”高性能计算平台，AI 平台。其中AI平台是老平台，用的是Tesla V100 GPU，而思源一号是 2022 年新上线的平台，用的是更新一代的 A100 40GB，根据 NVIDIA 官网给出的数据，性能远高于V100。同时因为新平台新上线的优势，大部分使用 A100GPU 是不用排队的，而很多电院的老师还在用AI平台，导致 AI 平台使用需要排队，所以强烈推荐大家使用思源一号平台。

平台连接

和学长沟通获取账号与密码/秘钥，秘钥连接更省心。建议安装 MobaXterm（可以记住连接各种设置，比如密码），Linux 系统直接写 Linux SSH Config 文件连接。连接上请在根目录下创建自己的文件夹，防止和他人冲突。

AI 平台连接域名：login.hpc.sjtu.edu.cn

思源一号

  • 连接域名：sylogin.hpc.sjtu.edu.cn
  • CPU 队列：64c512g
  • GPU 队列：a100

软件配置

我们常用的软件为 conda 和 python。

因为配置软件设计下载编译等操作，所以请先申请节点再进行软件配置，否则可能会被限制CPU使用率，严重可能加入黑名单无法使用集群。

conda 通过 module load miniconda3 加载，通过 source activate [env] 激活虚拟环境（不用执行 conda init ），通过 conda deactivate 退出虚拟环境。可以将 module load miniconda3 加入用户的 ~/.bashrc 中，省去加载步骤。之后使用 conda 安装 python 环境和平时使用是一样的。

申请节点/提交作业

可以通过脚本提交作业或者交互式使用资源，写代码调试时可以先申请 CPU 资源进行交互式作业，调试完成跑实验时可以再使用 GPU 队列。

思源一号 CPU 交互式作业（申请一个节点，4个核）：

srun -p 64c512g -N 1 -n 4 --pty /bin/bash

思源一号 GPU 交互式作业（申请一个节点，6个CPU核，1张 a100）：

srun -p a100 -N 1 -n 6 --gres=gpu:1 --pty /bin/bash

思源一号脚本提交 run.slurm (申请一个节点，6个CPU核，1张 a100）：

#!/bin/bash

#SBATCH --job-name=[your_job_name]
#SBATCH --partition=a100
#SBATCH -N 1
#SBATCH --ntasks-per-node=1
#SBATCH --cpus-per-task=6
#SBATCH --gres=gpu:1            #若使用 2 块卡，就给 gres=gpu:2
#SBATCH --output=%j.out
#SBATCH --error=%j.err

source activate your_conda_env
python [your_code.py]

思源一号 64c512g 脚本提交 (申请一个节点，只使用CPU核，6个）：

#!/bin/bash

#SBATCH --job-name=[your_job_name]
#SBATCH --partition=64c512g
#SBATCH -n 6
#SBATCH --output=%j.out
#SBATCH --error=%j.err

source activate your_conda_env
python [your_code.py]

常用命令

sinfo 查看总体资源情况：

$ sinfo
PARTITION AVAIL  TIMELIMIT  NODES  STATE NODELIST
cpu         up  7-00:00:0    656   idle cas[001-656]
dgx2        up  7-00:00:0      8   idle vol[01-08]

squeue 查看作业情况（常用指令，交互式执行跑完后一定要看看作业是否在执行，如果还在执行请用 scancel 取消作业）：

[seegxf-1@gpu06 ~]$ squeue
JOBID PARTITION     NAME     USER ST       TIME  NODES NODELIST(REASON)
16349782      a100     bash seegxf-1  R       0:02      1 gpu06

sacct 查看作业完成情况：

[seegxf-1@gpu06 ~]$ sacct -b
JobID             State ExitCode
------------ ---------- --------
16346958         FAILED      2:0
16346958.ex+  COMPLETED      0:0
16346958.0    COMPLETED      0:0

scancel JobId 取消作业

salloc 申请资源（提前申请资源并占据该资源 24 小时，需要分时段跑很多代码用这个比较适合，因为不用重新排队了，申请到的资源一直被你占用，不过跑完代码别忘了使用 scancel 取消资源，不然一直扣费！）：

salloc -p a100 -N 1 -n 6 --gres=gpu:1 -t 24:00:00

通过 squeue 查询到申请资源的节点名，通过 ssh 节点名 即可登录申请的节点。

$ salloc -p a100 -N 1 -n 6 --gres=gpu:1 -t 24:00:00
 salloc: Granted job allocation 16350463
 salloc: Waiting for resource configuration
 salloc: Nodes gpu01 are ready for job
$ squeue
JOBID PARTITION     NAME     USER ST       TIME  NODES NODELIST(REASON)
16350463      a100 interact seegxf-1  R       0:12      1 gpu01
$ ssh gpu01

工具

使用 Visual Studio Code，结合插件 python , Remote-SSH ，编写代码。vscode 中的终端默认登录的是登录节点，如果是交互式执行，需要手动切换到对应计算节点中去。比如通过 srun 申请到了节点 gpu06 ，在 vscode 终端里面 ssh gpu06 即可方便执行调试代码。
